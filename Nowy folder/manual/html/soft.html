The following programs from the author are included in the ERA pack:
— Tools\BinMagic
— Tools\Dee
— Tools\DEFka
— Tools\Evme
— Tools\ExeBuilder
— RepMan
— ResMan
— LodImport
— LodExport
— Objector
— DefExplode

The Tools\Erm Editor program by Alexander Trivonov was modified and now does not show comment tags when saving documents.

 

BinMagic ↑

Designated for work with binary patches in a format compatible with all versions of Era and the ExeBuilder utility. A technical description of the program parameters is given in the file "binmagichelp.txt". To simplify working in the catalog with the program, there are 3 bat files:
binary.bat, text.bat, diff.bat.

The first two take only one argument, the name of a patch file that will be optimized and translated into, respectively, a binary or text form. diff.bat serves to create patches through comparison of the original and changed files. It accepts three arguments:
- original file;
- changed file;
- patch save path.

For saving in binary form, the patch should have a "*.bin" extension; if not, then any other. All three bat files create memory patches which can be activated after they are placed in the catalog EraPlugins\[BeforeWoG|AfterWoG].

Text patch format:
The patch document consists of tags containing useful information (author, field of application, description), addresses to which the new data should be written, and the data, written in hex form (0-9, A-F).

Example:

; Author = Unknown ; Version = 1.0 ; Not for commercial use 00887668 12345678 00401FC7 9090A090B2

Addresses are separated from data with white spaces. Each new address should start on a new line.

 

Dee ↑

DEE means Dynamic ERM Editor.
This program permits the opening, editing, and (quick!) saving of game scripts, as well as their loading into the editor from game memory (00..99), loading from the editor to game memory, and deletion from game memory. There is also the option to unpack a script from a compressed form to a single line.


DEFka ↑

With one click of the mouse, this program allows the DEFs of any standard monster to be exchanged for others (of which there are many on the Internet) and restore the original files at any moment just as easily.

 

Evme ↑

ERM Variables Memory Editor is a utility that helps debug scripts, by allowing the user to view, change, or save to a file the values of most global ERM variables.

 

ExeBuilder ↑

A program for the creation of your own version of an executable file. Accepts all bin patches from folders to file In.exe, creating an Out.exe file upon exit.

Patches must be designated for files. BinMagic has the ability to transform patches for memory into patches for files and back.

 

RepMan ↑

Simple utility for replacing one line with another in the indicated file.

Example:

Data\s\script00.erm v2391 v600

 

ResMan ↑

Program for import, export, and creation of snd/vid files. Launch without parameters for help call-up.

 

LodImport ↑

Utility for importing a file to LOD archives.
Accepts two arguments:
- path to LOD archive, path to file.

 

LodExport ↑

Utility for exporting files from LOD archives.
Accepts three arguments:
- path to archive, file name in archive, path to destination file.

 

Objector ↑

Allows the files for objects (z[e]obj[c]ts.txt) to be correctly unified with user files.
Arguments:
- path to main txt file, path to installable file, [option] add information on new objects without replacing old ones.

 

DefExplode ↑

Separates indicated DEF into frames in bmp format.
Arguments:
- path to DEF, folder for content.

The use of the programs LodImport and ResMan in a pair with bat files allows the complete avoidance of installer WoG — h3wupd.exe, which is very often unstable in its operations.
